Tabella sync_run

Metadati del job di sincronizzazione. Ogni sync elimina i run precedenti e ricrea lo snapshot issue.

Campo Tipo SQLite Null Descrizione
PKid TEXT (UUID) NO Identificatore univoco del job di sync.
started_at DATETIME NO Timestamp avvio scarico; default now().
finished_at DATETIME SÌ Fine job; NULL mentre status = running.
source TEXT NO Origine dati; default jira-api.
issue_count INTEGER NO Numero issue scritte in jira_issue in questo run.
status TEXT NO Stato job: running, success, partial, failed.

Tabella jira_issue

Cache backlog allineata a JiraBacklogRow (cruscotto.frontend/cruscotto.jira.backlog.mjs). Popolata da cruscotto.database/Jira.backlog.sync.mjs.

Campo Tipo SQLite Null Descrizione
PKid TEXT (UUID) NO Chiave interna riga cache (non è la key Jira).
UKjira_key TEXT NO Chiave Jira (ADMIN-154, JLO-507); unique su tutta la tabella.
issue_type TEXT NO Tipo Jira: Story, Bug, Sub-task, Epic, …
summary TEXT NO Titolo issue.
status TEXT NO Stato workflow Jira (es. Da fare, In corso, Fatto).
status_category TEXT SÌ Categoria Jira: new, indeterminate, done.
parent_jira_key TEXT SÌ Key parent (subtask → story); indice su questa colonna.
jira_updated_at DATETIME SÌ Ultimo aggiornamento lato Jira (se disponibile nel sync).
tier TEXT NO Livello albero backlog: epic, task, subtask.
is_story_like BOOLEAN NO true per Story/Bug/Todo trattati come unità di sviluppo (gogo/PUSH).
is_done BOOLEAN NO true se stato in categoria «done» (isJiraStatusDone).
depth INTEGER NO Profondità nell’albero epic → story → subtask; default 0.
has_children BOOLEAN NO true se l’issue ha figli nel backlog.
dev_order TEXT SÌ Ordine dev gerarchico (es. 1.2.3) da working plan.
dev_sprint INTEGER SÌ Numero sprint di sviluppo.
dev_sprint_name TEXT SÌ Nome sprint di sviluppo.
dev_sort INTEGER SÌ Ordinamento numerico all’interno dello sprint.
is_sprint6_obsolete BOOLEAN NO Flag issue obsolete nello sprint 6; default false.
related_keys TEXT (JSON) SÌ Array JSON di chiavi correlate (issue links); es. ["ADMIN-12","JLO-3"].
raw_fields TEXT (JSON) SÌ Payload esteso opzionale; raramente popolato in sync standard. Vedi raw_fields.
FKsync_run_id TEXT NO Riferimento a sync_run.id; cascade delete con il run.
synced_at DATETIME NO Timestamp scrittura riga in cache; default now().

Tabella jira_issue_wip

Coda WIP — stessa struttura colonne di jira_issue, ma solo ticket nel workflow locale (veve db, gogo MyBacklog, chiudi WIP, PUSH). Una riga per jira_key in lavorazione.

I campi sotto sono identici a jira_issue salvo il ruolo semantico: qui raw_fields è il cuore del workflow database (veve, git, PUSH).
Campo Tipo SQLite Null Descrizione
PKid TEXT (UUID) NO Chiave interna riga WIP.
UKjira_key TEXT NO Chiave Jira; unique — un solo record WIP per ticket.
issue_type TEXT NO Tipo Jira al momento dell’iscrizione in coda.
summary TEXT NO Titolo issue.
status TEXT NO Stato workflow; aggiornato da ok chiudi / finalize gogo.
status_category TEXT SÌ Categoria stato Jira.
parent_jira_key TEXT SÌ Parent per subtask in coda WIP.
jira_updated_at DATETIME SÌ Ultimo aggiornamento Jira noto.
tier TEXT NO epic · task · subtask.
is_story_like BOOLEAN NO Story-like per UI gogo/PUSH.
is_done BOOLEAN NO true dopo step 7 chiudi WIP o finalize gogo con sviluppo completo.
depth INTEGER NO Profondità albero backlog.
has_children BOOLEAN NO Ha subtask nel backlog.
dev_order TEXT SÌ Ordine dev da working plan.
dev_sprint INTEGER SÌ Numero sprint dev.
dev_sprint_name TEXT SÌ Nome sprint dev.
dev_sort INTEGER SÌ Sort in sprint.
is_sprint6_obsolete BOOLEAN NO Flag obsolescenza sprint 6.
related_keys TEXT (JSON) SÌ Chiavi correlate (array JSON).
raw_fields TEXT (JSON) SÌ Stato workflow, veve, git, PUSH — vedi sezione dedicata.
FKsync_run_id TEXT NO FK al sync_run dell’ultimo sync usato in enroll.
synced_at DATETIME NO Ultimo aggiornamento riga WIP.

API matrice cruscotto

Endpoint REST su cruscotto.server.mjs (porta dashboard) — story ADMIN-173. Consumer: toolbar docs «Aggiorna analisi», agenti, future tab Avanzamento.

Endpoint Metodo Descrizione
GET /api/matrix/portal-gap GET Sezioni + righe correnti da matrix_row · query ?kind=portal_gap
GET /api/matrix/runs GET Storico snapshot matrix_run · ?kind= · ?limit=20
GET /api/matrix/rows/:findingId/events GET Audit trail append-only matrix_row_event per voce
POST /api/matrix/finding-issue POST Body JSON { findingId, key, issueType? } → upsert matrix_finding_issue
POST /api/matrix/regenerate POST Scan repo + upsert DB + HTML (equivalente CLI) · solo localhost

Modulo servizio: cruscotto.database/matrix.api.service.mjs · test: admin.portal.testscript/technical/test.matrix.api.e2e.mjs

Colonna raw_fields (JSON)

Campo TEXT con oggetto JSON serializzato. Usato soprattutto in jira_issue_wip; in jira_issue resta opzionale/vuoto nel sync standard. Merge incrementale: ogni step workflow aggiorna solo le chiavi che tocca.
