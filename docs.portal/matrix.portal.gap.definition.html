Flusso esecuzione

matrix.portal.gap.analysis.mjs
  └─ runAnalysisProcedure({ portalRoot })
       ├─ step 1: analyzePortalAdvancement(portalRoot)  → report + metrics
       ├─ step 2: buildGapMatrixSections(report)        → MatrixSection[]
       ├─ step 3: writeGapMatrixArtifacts(...)          → .json + .md
       └─ step 4: generateGapMatrixHtml(report)         → HTML (solo in procedura isolata)
  └─ runFullGapAnalysis
       ├─ writeGapMatrixArtifacts (sempre)
       └─ HTML: merge su matrix.portal.gap.html esistente | full render (--full)

CLI e flag

# Rigenerazione completa (default: merge HTML se esiste)
node docs.portal/matrix.portal.gap.analysis.mjs

# Render HTML completo senza merge (sovrascrive stati Issue refinement in pagina)
node docs.portal/matrix.portal.gap.analysis.mjs --full

# Emette HTML su stdout (utile per pipe o ispezione)
node docs.portal/matrix.portal.gap.analysis.mjs --stdout-html
FlagEffettoExit
(nessuno) Scrive JSON, MD, HTML; merge su HTML esistente se presente 0 · 1 su errore
--full Render completo via generateGapMatrixHtml 0 · 1 su errore
--stdout-html Stampa HTML su stdout; non stampa riepilogo path file 0 · 1 su errore

Stdout di successo (senza --stdout-html): path file scritti + riga metriche (Arch N%, sezioni, gap, bug).

Modalità merge vs full

ModalitàCondizioneComportamento
Merge (default) HTML esiste e non è passato --full Legge stati Issue refinement precedenti, aggiorna sezioni e metriche, rimuove mark «fresh»
Full render --full o HTML assente Rigenera pagina da zero con renderMatrixPage

Evitare edit manuale solo dell'HTML output: al prossimo merge i cambi non strutturati possono andare persi. Preferire rigenerazione da script.

Categorie e severità

Sezioni nella matrice HTML:

avanzamento · architettura · gap · bug · deprecation · feature · miglioramento

Severità finding: P0 · P1 · P2 · info

Area analizzataMetodo (indicativo)

Gap analyzeRepository + import live da PARKING
Bug Grep import mancanti, path CI, consumer npm
Deprecation Drift commenti runner, stub start-dev
Feature / miglioramenti Catalogo curato in matrix.gap.scan.mjs
