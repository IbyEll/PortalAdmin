In PortalAdmin esiste già un sistema di sanity check orientato al progetto, non un generico linter tipo ESLint o Knip. È pensato per architettura, ridondanze, drift, gap e miglioramenti — non per rilevare automaticamente ogni export mai importato.

3. Visualizzazione in UI

Con npm run admin:home attivo:

  1. Apri la HOME portal → tab Documenti
  2. Apri matrix.portal.gap.html dall'indice
  3. Usa Rigenera sulla toolbar per rilanciare lo scan

In alternativa, dal cruscotto in dev locale:

  • POST /api/matrix/regenerate — equivalente API con persistenza su DB matrice
  • POST /api/docs/regenerate — rigenera singola pagina matrice dalla HOME

5. Cosa non copre oggi

Nel repository non risultano configurati strumenti generici per:

  • export o funzioni mai importate (es. Knip, ts-prune)
  • dipendenze npm inutilizzate (depcheck)
  • linting statico (ESLint)

La matrice PortalAdmin è rule-based + grep + catalogo curato: ottima per drift architetturale e ridondanze note, meno per «tutto il codice morto» in automatico.

6. Workflow consigliato per un sanity check completo

1. node docs.portal/matrix.portal.gap.analysis.mjs
2. Apri matrix.portal.gap.html → sezioni audit-* e #audit-parking
3. node docs.portal/matrix.test.coverage.mjs
4. npm run test:ci
5. (opzionale) npm run backlog:obsolete -- --format md

In chat Cursor puoi anche chiedere all'agente: analizza avanzamento — attiva la regola ADMIN-AvanzamentoGapFeature che rigenera e interpreta la matrice.
