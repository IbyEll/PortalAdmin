PortalAdmin — Visione sviluppo

Documento: 20260620_PortalAdmin_development.html · 20 giugno 2026 · repo IbyEll/PortalAdmin

PortalAdmin è un hub di tooling dev: cruscotto browser, cache Jira, runner testScript, workflow Cursor. Può puntare a product repo diversi tramite overlay PROJECT_{PRJ_NAME} e variabile PRODUCT_REPO_PATH.

1. Dual-repo e overlay

Il cruscotto legge config, discovery, testScript e (opzionalmente) Prisma dal product repo attivo. La HOME (:3990) istanzia overlay e avvia cruscotto su porta dedicata.

7. Comandi rapidi

npm install
cp .env.example .env          # PRJ_NAME, PRODUCT_REPO_PATH, Jira token

npm run admin:home            # HOME :3990 — scelta overlay
npm run admin:dashboard       # Cruscotto (PRJ_DASHBOARD_PORT)

npm run dev:all               # Stack product JLO (web+api+auth)
npm run test:portal-api       # Smoke API cruscotto
npm run test:run-all          # Discovery testScript
npm run db:sync               # Sync backlog Jira → SQLite

Variabili env principali

PRODUCT_REPO_PATHRoot monorepo product (default sibling)
DASHBOARD_PORTPorta cruscotto (3998 / 3999)
PORTAL_HOME_PORTPorta HOME (default 3990)
JIRA_*Token e cloud per sync backlog
