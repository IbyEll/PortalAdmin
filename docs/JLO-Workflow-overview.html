JLO-Workflow — Panoramica

Regola Cursor: .cursor/rules/JLO-Workflow.mdc · Generato 2026-06-15 · alwaysApply: true

Orchestra git + Jira + codice + commit + PR sui ticket JLO-xxx (Story, Bug, Todo) con un protocollo chat rigido: START → PROGRESS → END, senza dump Jira grezzo in chat.

Scopo generale

JiraPiano, implementazione, chiusura subtask e parent
ChatSolo START, PROGRESS, END (+ blocker)
ToolingJira via subagent Task «Jira Auto»; chiusura via close-story.mjs

Ordine fisso all'avvio: branch → pre-flight git → Step 0 → implementazione. FULL cambia solo come si fa §7 (confirm vs silent), non salta §4.

Git — regole principali (§1, §4, §5)

Naming branch

  • Story → STORY---JLO-{key}-{slug}
  • Bug → BUG---JLO-{key}-{slug}
  • Todo → TODO---JLO-{key}-{slug}
  • Legacy (solo ripresa): JLO-{key}-*

Vincoli

  • Mai lavorare/commitare su main
  • Working tree pulito prima di aprire un ticket (no stash automatico)
  • Branch non deve essere ahead rispetto a origin senza gestione
  • Una sola branch per key+tipo

Step 0 — Piano (§6)

Prima del codice, in background:

  1. Task Jira Auto — legge parent + subtask
  2. Gap analysis repo — JLO-AnalizaRepo.mdc + CLI analyze-repo-keys.mjs
  3. Filtra subtask Da fare, ordina (schema → API → web → test → manuale)
  4. Estrae AC/DoD nel template START (mai copiare description Jira)

Output chat unico: blocco START (§10.1). Contesti Jira conservati in sessione per ok chiudi / chiudi.

Implementazione (§7) e gate ok chiudi (§8)

Una subtask alla volta. Sequenza obbligatoria a fine subtask:

git add + commit  →  editJiraIssue (Stato repo + DoD)  →  transition Fatto  →  commento Jira
ModalitàDopo implementazione
Confirm (default)Silenzio fino a «ok chiudi», poi §8
Silent (gogo, FULL silent)§8 subito, senza attendere

Se commit o Jira falliscono → stop flusso, END parziale con Errori:. Push non automatico dopo ogni subtask (solo in chiudi o opzionalmente dopo gogo).

Jira — invocazione (§5bis)

Tutte le operazioni Jira passano da Task subagent (skill jlo-jira-auto), mai MCP diretto dall'agente principale.

FaseTask description
Step 0Step 0 JLO-{key} plan Jira
Fine subtaskOk chiudi JLO-{subkey} subtask Jira
Chiusura parentClose JLO-{key} parent Jira

Chiusura parent (§9 / §9bis)

Script unico:

node Admin/scripts/close-story.mjs --key JLO-{key}

Verifica tree pulito → aggiorna catalogo segnali repo → push branch → apre PR su main. Poi Task Jira chiude il parent (subtask devono essere già Fatto).

VarianteExtra
chiudi completoGap test + subtask successiva → END-B
chiudi fastSolo git/PR/Jira parent → END-B-FAST

L'agente non fa merge PR: lo fa l'utente su GitHub.

Protocollo chat (§0, §10)

Tra START e END l'agente non può narrare («Ora leggo…», diff, path, JSON Jira, ecc.).
