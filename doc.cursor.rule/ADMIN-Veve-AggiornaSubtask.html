ADMIN-Veve — Aggiorna Subtask

.cursor/rules/ADMIN-Veve-AggiornaSubtask.mdc · veve step 4

Per ogni subtask del parent ADMIN-xxx aggiorna la description Jira con obiettivo, stato repo, AC/DoD e file coinvolti derivati dalla gap analysis.

Trigger

  • aggiorna subtask ADMIN-xxx
  • veve ADMIN-xxx (step 4 finale)

Applicazione step-by-step

  1. Itera subtask — elenco da step 1, ordine N/M da story parent (step 3).
  2. Apri template — .cursor/templates/veve/veve-subtask.md per ogni key.
  3. Compila da gap analysis — Obiettivo, Stato repo, AC/DoD (B poi A), file coinvolti, ordine N/M.
  4. editJiraIssue — una chiamata per subtask (solo description).
  5. Riepilogo veve — tabella subtask aggiornate; grooming completato.
