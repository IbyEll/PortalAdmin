ADMIN-Veve — Leggi Jira

.cursor/rules/ADMIN-Veve-LeggiJira.mdc · veve step 1

Lettura read-only del parent ADMIN-xxx e delle subtask collegate — primo passo della sequenza veve.

Trigger

  • leggi jira ADMIN-xxx
  • veve ADMIN-xxx (step 1 automatico)

Applicazione step-by-step

  1. Invoca MCP — CallMcpTool server user-atlassian, cloudId 3caddd74-469e-4ca3-adf8-926f79c98e7c.
  2. Leggi parent — key, issuetype, stato, summary.
  3. Elenca subtask — tabella con key, summary, stato.
  4. Segnala anomalie — subtask orphan (senza parent), key invalide o fuori scope.
  5. Output chat — riepilogo strutturato per step 2 (Analizza repo).
