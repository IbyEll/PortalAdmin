Workflow — progetto istanziato + {JIRA_KEY}

Il workflow git (branch, commit, push, PR) opera solo sul repo del progetto istanziato, non sul checkout PortalAdmin in sé.

RuoloPath / env
Host toolingCheckout PortalAdmin — cruscotto, regole .cursor/, script admin.portal.JiraCORE/
Repo workflowPRODUCT_REPO_PATH del progetto istanziato (PRJ_NAME in .env allineato)
Default pathSibling ../{PRJ_REPO} da PROJECT_{PRJ_NAME}/project.config.{overlay}.mjs

Esempi: overlay JustLastOne → repo workflow …/JustLastOne · overlay AdminDashBoard → …/PortalAdmin (product = stesso repo).

Prerequisito — istanziazione

Prima di procedi / gogo / chiudi su {KEY}:

ControlloStop se
PRJ_NAMEAssente o non corrisponde all'overlay del ticket
PRODUCT_REPO_PATHAssente, path inesistente o non allineato all'istanza portal
JIRA_KEY del ticketDiverso da PRJ_JIRA_PREFIX del progetto istanziato
Cwd gitNon è la root del repo workflow (git rev-parse --show-toplevel ≠ PRODUCT_REPO_PATH)

Istanza portal: activatePortalInstance / HOME POST /api/portal/instance · stato in .env (PRJ_NAME, PRODUCT_REPO_PATH).

Non avviare workflow su ticket il cui prefisso Jira non appartiene al progetto attualmente istanziato (es. JLO-507 con solo overlay AdminDashBoard e product non puntato a JustLastOne).

Verifica branch (step 1)

Eseguire nel repo workflow (cd su PRODUCT_REPO_PATH):

git rev-parse --show-toplevel   # deve coincidere con PRODUCT_REPO_PATH
git branch --show-current
git branch --list 'STORY---{JIRA_KEY}-{num}-*'
git branch -r --list 'origin/STORY---{JIRA_KEY}-{num}-*'
# Idem BUG--- / TODO---
SituazioneAzione
Non sul repo workflowStop — cd su PRODUCT_REPO_PATH o istanzia overlay
Sei sulla branch del ticketRipresa
Branch esiste, sei altrove (stesso repo)Stop — git checkout {branch} e ripeti procedi
Nessuna branchPre-flight → nuova branch da main

Pre-flight git (step 2)

Nel repo workflow (PRODUCT_REPO_PATH):

  1. git status --porcelain — vuoto (no stash automatico agente)
  2. git fetch origin · main allineato a origin/main
  3. Nessuna PR aperta sul repo workflow — gh pr list --state open; se ≥1 → Stop (merge o chiudi prima di gogo / procedi)
  4. git checkout main && git pull && git checkout -b {TIPO}---{JIRA_KEY}-{num}-{slug} (solo se nuova branch)

Runtime: checkNoOpenPullRequests() in admin.portal/portal.cursor.agent.workflow.mjs · API GET /api/workflow/gogo-preflight.

Step 3 — Veve DB (grooming → jira_issue_wip)

Obbligatorio su procedi / gogo su {KEY} — dopo pre-flight git, prima di Step 0.

Esegui regola ADMIN-Veve.db.mdc (e sotto-regole ADMIN-Veve.db-*.mdc) su {KEY}:

Sotto-stepRegola
3.1ADMIN-Veve.db-LeggiJira.mdc — leggi parent + subtask da jira_issue
3.2ADMIN-AnalizzaRepo.mdc — gap analysis repo workflow
3.3ADMIN-Veve.db-AggiornaStory.mdc — upsert parent in jira_issue_wip
3.4ADMIN-Veve.db-AggiornaSubtask.mdc — crea subtask test mancanti + upsert ogni subtask in jira_issue_wip
AspettoRegola
ChatSilenzioso — ADMIN-Silence.mdc; output veve solo in sessione per Step 0
JiraVietato MCP — lettura cache DB; scrittura solo jira_issue_wip
Git / codiceVietato — no commit, no branch, no implementazione
Prerequisitojira_issue popolata (npm run db:sync) per {KEY}
Standaloneveve db {KEY} — chat libera, stessa sequenza

Ripresa: se veve db già eseguito nella stessa sessione su {KEY} → salta step 3.

Step 0 — Piano (step 4)

Dopo veve db, prima dell'implementazione — legge DB, non Jira live:

  1. Leggi WIP — jira_issue_wip parent {KEY} + subtask (parent_jira_key = {KEY}) via openCruscottoDb() / Prisma jiraIssueWip
  2. Piano — estrae da raw_fields JSON (veveDescription, gapSummary, AC/DoD compilati in step 3.3–3.4); non Task «Jira Auto» · non MCP
  3. Gap — riusa output step 3.2; non ripetere CLI gap salvo ripresa senza veve db in sessione
  4. Filtra subtask Da fare (is_done, status da righe WIP), ordina (schema → API → web → test → manuale)
  5. Compila template START da WIP + gap (mai description Jira grezza)

Output chat unico: blocco START (ADMIN-Silence.mdc).

Stop se: parent assente in jira_issue_wip → rieseguire step 3 veve db.

Step 5 — Implementazione + ok chiudi (WIP)

Una subtask alla volta. Sequenza obbligatoria a fine subtask {subkey}:

  1. Codice — implementazione nel repo workflow
  2. Commit — git add + commit messaggio {subkey} … (repo workflow)
  3. Ok chiudi — chiusura subtask su jira_issue_wip, non Task «Jira Auto» · non MCP Jira

Ok chiudi — update jira_issue_wip

Upsert/update riga jira_key = {subkey}:

CampoValore
is_donetrue
statusFatto (o equivalente chiuso)
raw_fieldsMerge JSON: statoRepo, doD, commitHash, closedAt (ISO), aggiorna veveDescription se serve
synced_atnow

Prisma: db.jiraIssueWip.update({ where: { jiraKey: subkey }, data: { … } }).

Template testo chiusura (opzionale in raw_fields): .cursor/templates/workflow/ok-chiudi-subtask.md — compilato da gap post-commit, senza editJiraIssue.

ModalitàDopo implementazione
Confirm (default procedi)Silenzio fino a «ok chiudi», poi update WIP
Silent (gogo, FULL silent)Update WIP subito dopo commit

Stop se: subtask assente in jira_issue_wip → stop flusso, END parziale con Errori:.

Push non automatico dopo ok chiudi — push/PR solo in step 8 PUSH.

Step 7 — Chiusura locale (solo WIP)

Su chiudi Story {KEY} / chiudi fast — dopo END-A, tutte le subtask Fatto in WIP:

  1. Verifica WIP — ogni subtask {subkey}: is_done = true; stop altrimenti
  2. Gap test (solo chiudi completo, non chiudi fast) — ADMIN-AnalizzaRepo.mdc
  3. Chiudi parent WIP — update riga jira_key = {KEY}:
CampoValore
is_donetrue
statusFatto
raw_fieldsMerge JSON: chiudiParent, gapTest (se applicabile), wipClosedAt (ISO), awaitingPush: true
synced_atnow

Niente push git · niente PR · niente Jira live · niente close-story.mjs.

  1. END-B / END-B-FAST — template workflow; PR: — (PR allo step 8)
  2. Ritorno backlog cruscotto — aggiorna riga {KEY}: stato WIP chiuso, pulsante GOGO → PUSH

Template testo locale (in raw_fields): .cursor/templates/workflow/chiudi-parent.md — senza prUrl.

Step 8 — PUSH (Atlassian + cache jira_issue + PR)

Solo cruscotto — pulsante PUSH sulla riga backlog dopo step 7.

Prerequisito: parent {KEY} in jira_issue_wip con is_done = true e raw_fields.awaitingPush = true.

  1. Leggi WIP — parent + subtask
  2. Sync Atlassian — description ADF + transizione Fatto (jiraCORE.jira.live.mjs, REST)
  3. Git / PR — close-story.mjs: catalogo → push branch → PR
  4. Update WIP — markWipPushed: prUrl, pushedAt, awaitingPush: false (riga WIP resta)
  5. Sync jira_issue — syncJiraIssueCacheFromWip con purgeIfAligned: false

Step 9 — PR merge → purge WIP

Dopo step 8, poll PR (POST /api/jira/wip/pr-poll · jiraCORE.wip.pr.poll.mjs).

EventoAzione
PR OPENAggiorna prLastPolledAt in WIP; riga WIP resta
PR MERGED / CLOSEDSync jira_issue → elimina parent + subtask da jira_issue_wip (purgeWipBundleIfCacheAligned)

Purge WIP solo se cache allineata (verifyWipBundleAlignedWithCache) e prPollComplete / prMergedAt valorizzati.
