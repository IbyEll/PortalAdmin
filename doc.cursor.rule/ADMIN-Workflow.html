ADMIN-Workflow

.cursor/rules/ADMIN-Workflow.mdc · alwaysApply: true · Aggiornato 2026-06-22

Orchestra git, Jira e codice sui ticket del progetto istanziato (PRJ_NAME + PRODUCT_REPO_PATH), non sul solo checkout host PortalAdmin.

Revisione 2026-06-22: integrato ADMIN-Veve come step 3 obbligatorio su procedi / gogo, prima di Step 0.

Pipeline — ordine fisso

Prima (superato)
  1. Verifica branch
  2. Pre-flight git
  3. Step 0 — Task Jira Auto + gap analysis CLI + START
  4. Implementazione subtask
  5. END-A → chiudi → END-B
Adesso

2Pre-flight gittree pulito, branch da main
3VeveADMIN-Veve — grooming Jira silenzioso
4Step 0 — pianoTask «Jira Auto» + START (Silence)

gogo / FULL non saltano gli step 1–4. chiudi non riesegue veve.

Eseguito automaticamente su procedi / gogo {KEY} dopo pre-flight, prima di Step 0.

Step 0 — Piano (step 4)

Prima

Step 0 eseguiva Task Jira Auto e gap analysis CLI in parallelo al primo avvio.

Adesso
  1. Task «Jira Auto» — Step 0 {KEY} plan Jira su Jira già allineato da veve
  2. Gap — riusa output step 3.2; non ripetere CLI salvo ripresa senza veve in sessione
  3. Filtra subtask Da fare, ordina, estrae AC/DoD
  4. Emissione unica START in chat

Comandi utente

ComandoEffetto

procedi Story {KEY}Step 1–4 → implementazione con confirm
chiudi Story {KEY}Push + PR + catalogo + gap test + parent → Fatto
veve {KEY}Solo grooming (standalone, chat libera) — stessa sequenza dello step 3
