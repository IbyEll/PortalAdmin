ADMIN-Veve — Aggiorna Story

.cursor/rules/ADMIN-Veve-AggiornaStory.mdc · veve step 3

Aggiorna la description del ticket parent ADMIN-xxx allineandola allo stato repository emerso dalla gap analysis.

Trigger

  • aggiorna story ADMIN-xxx
  • veve ADMIN-xxx (step 3, dopo Leggi Jira + Analizza repo)

Prerequisiti

  • Output Leggi Jira (parent + subtask)
  • Output Analizza repo (Stato repo, gap)

Applicazione step-by-step

  1. Apri template — .cursor/templates/veve/veve-story-parent.md.
  2. Compila placeholder — Obiettivo, Stato repo, AC/DoD (checkbox B poi A), ordine subtask dai dati Jira + gap analysis.
  3. Verifica sezioni — formato canonico JLO-Veve-AggiornaStory per checkbox AC/DoD.
  4. editJiraIssue — aggiorna solo description del parent.
  5. Conferma in chat — parent aggiornato; passa a step 4 se in sequenza veve.

Campi description parent

SezioneFonte dati
ObiettivoSummary + contesto Jira
Stato repoGap analysis step 2
AC / DoDAllineati al repo; checkbox B poi A
Ordine subtaskSequenza implementazione suggerita

Vietato

  • Modificare subtask (step 4)
  • Branch, commit
  • Transizione parent a Fatto
